La Terminal (Shell)

2016-08-18

Shell, Terminal, Linea de comandos?

La terminal es una aplicación que viene pre-instalada en los sistemas basados en unix que siguen el estándar POSIX y nos brinda un ambiente de entradas/salidas que “contiene” o comunica al shell los comandos que escribimos en ella para que sean interpretados y nos muestre el resultado de salida en pantalla.

Existen diferentes shells disponibles que podemos instalar entre los más usados están bash y zsh, yo voy a utilizar zsh. Para fines prácticos nos vamos a referir a la terminal de ahora en adelante.

Nota: windows cuenta con cmd (linea de comandos) y powershell que cumplen propósitos similares pero siguen arquitecturas diferentes al shell de unix. Por eso debemos instalar un shell compatible con ayuda de Cygwin o en el caso de Windows 10 ya pueden activar bash como un feature del sistema.

Por qué aprender a usar la terminal?

Bueno aquí unas cuantas razones:

  • Mucho del workflow para desarrollar aplicaciones web se basan en pequeñas herramientas que corren en la terminal, por ejemplo: gulp, webpack, node, npm, bower, stylus etc.

  • El control de versiones con git también se hace a través de la terminal, hay herramientas con GUI pero para sacarle provecho a git siempre terminaremos yendo a la terminal.

  • La conexión y el manejo de servidores Linux remotos se hace a través de la terminal usando SSH.

  • Las herramientas de seguridad informática o hacking también corren en terminal, así que si quieren ser cómo Mr. Robot (en lo técnico no necesariamente en la esquizofrenia y las drogas…) aprendan a usar la terminal.

Árbol de directorios

El árbol de directorios es la representación estructurada de la información que contiene nuestro disco duro. Ejecuten la terminal y esto es lo primero que haremos:

$ pwd Nos da el path del working directory actual, es decir donde estamos situados en el árbol de archivos.

$ ls Lista el contenido del directorio en el que nos encontramos

Nota: $ clear Sirve para limpiar la pantalla

ls funciona bien pero podemos sacarle más provecho agregando algo llamado banderas

$ ls -l Impreme con Formato largo

$ ls -a Imprime incluyendo los archivos ocultos

Y por supuesto podemos mezclar banderas $ ls -l -a o simplemente $ ls -la

No hay un estándar en cuanto a la forma de enviar las banderas siempre consulta el manual del comando para saber cómo utilizar sus banderas

$ man ls Al comando man le mandamos como argumento el nombre del comando del cual queremos leer su manual. Se presiona Enter para avanzar en el documento y q para salir.

Normalmente la gran mayoría de comandos tienen un manual que se accede con man pero hay ciertos comandos de terceros que hacen uso de la bandera -h o en su defecto --help para mostrar la ayuda de como deben ser utilizados, por ejemplo: nvm -h o git --help.

Ahora que ya sabemos cómo listar el directorio en detalle, veamos las columnas que nos muestra

permisos | hardlinks-count | username and group | size | modified | filename/dirname

  • permisos
    Sobre la columna de permisos también se muestra con el primer carácter si se trata de un directorio, archivo o link.

    d directorio drw-r--r--
    l = link lrw-r--r--
    - = archivo -rw-r--r--

El resto tiene que ver con permisos que abordaremos en otro post.

  • hardlinks count Es un conteo de los archivos que contiene cada folder más la referencia a su directorio padre .. y a si mismo ., en el caso de archivos el valor es 1 (más sobre soft y hard links en otro post).

  • username and group Nombre de usuario dueño del archivo y el grupo al que pertenece.

  • size Tamaño del directorio o archivo en bytes.

  • modified Ultima fecha de modificación.

  • filename/dirname Nombre del archivo o directorio.

Moverse entre directorios

Para avanzar un subdirectorio utilizamos el comando change directory cd [subdirectorio]

Hay tres formas de volver al $HOME que sería el directorio raíz del usuario actualmente en el sistema.

  • $ cd $HOME
  • $ cd ~
  • $ cd

Subir un nivel, ir al directorio padre

Estando en /Users/elyager/Documents/GitLab/Enki

$ cd .. nos deja en /Users/elyager/Documents/GitLab/

Subir dos niveles? fácil

$ cd ../..

Crear, Cortar, Pegar diretorios y archivos

$ touch “Toca” una archivo y si no existe simplemente cambia su ultima fecha de modificación

$ mkdir [DIRNAME] Crea un directorio nuevo

$ ln -s Soft Link y Hard Links (post aparte)

$ mv Mueve de un lado a otro (cortar y pegar). Sirve también para renombrar

Mover un archivo a un subdirectorio y después regresarlo a la directorio actual

Mover un directorio y su contenido

Recuerden . hacer referencia a nuestro directoria actual.

Usar mv para renombrar

$ cp Copia de un lado a otro

Para copiar un directorio con todo su contenido hay que agregar la bandera -r para hacerlo de forma recursiva.

Con esto abarcamos la navegación y creación de directorios, recuerden que siempre pueden profundizar en cada uno de estos comandos leyendo el manual con man, sé que esto puede ser tedioso pero una vez que se domina es muchisimo más rápido que utilizar el explorar de archivos gráfico del sistema se los aseguro.😉
